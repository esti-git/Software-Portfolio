# LiveTickets 🎫

LiveTickets היא מערכת שלמה (Full-Stack) המאפשרת למשתמשים לדפדף באירועים והופעות חיות, לסנן לפי קטגוריות, לנהל סל קניות מבוסס מוצרים ולבצע רכישת כרטיסים מאובטחת. 

המערכת בנויה בארכיטקטורת שכבות מתקדמת בצד השרת וממשק משתמש דינמי ומודרני בצד הלקוח.

---

## 🚀 טכנולוגיות מרכזיות

### Frontend (צד הלקוח)
* **Angular** - פיתוח קומפוננטות ריאקטיביות וממשק משתמש מודרני.
* **Vite** - כלי בנייה (Build Tool) מהיר לניהול השרת המקומי לפיתוח.
* **RxJS** - ניהול זרימת נתונים אסינכרונית וקריאות HTTP לשרת.

### Backend (צד השרת)
* **C# / .NET Web API** - בניית שרת ה-API המרכזי.
* **ארכיטקטורת שכבות (Layered Architecture):**
  * **WebApp** - שכבת ה-API, ה-Controllers והגדרות השרת.
  * **BLL (Business Logic Layer)** - שכבת הלוגיקה העסקית של האפליקציה.
  * **DAL (Data Access Layer)** - שכבת הגישה הנגישה ישירות לבסיס הנתונים (Repository).
  * **DTO (Data Transfer Object)** - שכבה האחראית על העברת נתונים מותאמת אישית בין השכבות השונות וצד הלקוח.

### Database (בסיס הנתונים)
* **MS SQL Server** - ניהול מסד הנתונים היחסי (Relational DB) של המערכת.

## 💻 פיצ'רים מרכזיים במערכת

* **מערכת משתמשים מלאה:** תמיכה בתהליכי הרשמה (Register) והתחברות (Login) מאובטחים עבור לקוחות.
* **דף הבית וקטלוג אירועים:** תצוגה עשירה של כל ההופעות והאירועים הזמינים במערכת, כולל שליפת תמונות דינמיות משרת ה-Backend.
* **סינונים וחיפושים מתקדמים:** אפשרות סינון אינטראקטיבית של האירועים לפי קטגוריות שונות בזמן אמת.
* **עמוד פרטי אירוע (Detail View):** בלחיצה על אירוע, המשתמש מועבר לעמוד המציג את כל המידע המלא: תאריך ושעה, מיקום, מגבלת גיל, החברה המפיקה ומחיר.
* **ניהול סל קניות (Cart Management):**
  * הוספת כרטיסים לאירועים ישירות לסל הקניות.
  * הסרה ומחיקה של אירועים מהסל.
  * עדכון כמויות וחישוב אוטומטי של הסכום הכולל לתשלום.
* **קופה ותשלום (Checkout):** תהליך מעבר לתשלום הכולל ממשק ייעודי להזנת פרטי אשראי והשלמת ההזמנה בבסיס הנתונים.

## 🗄️ מבנה בסיס הנתונים (Database Schema)

מסד הנתונים נקרא `EventStore` והוא מורכב מ-6 טבלאות המקושרות ביניהן:

* `Categories` - קטגוריות של אירועים (למשל: הופעות חיות, סטנדאפ, הצגות).
* `Companies` - החברות המפיקות והמארגנות את האירועים.
* `Events` - האירועים עצמם (כולל תאריך, מיקום, מחיר, קטגוריה וחברה).
* `Customers` - פרטי המשתמשים הרשומים במערכת (שם משתמש, אימייל וסיסמה).
* `Purchases` - טבלת הזמנות ראשית (קישור ללקוח, תאריך הרכישה וסכום כולל).
* `PurchaseDetails` - פירוט השורות של כל הזמנה (איזה כרטיסים נרכשו, כמות ומחיר ליחידה).

## 🛠️ הוראות הפעלה והרצה

### דרישות קדם
* .NET SDK (גרסה 8 ומעלה)
* Node.js & Angular CLI
* SQL Server Management Studio (SSMS)

### 1. הגדרת בסיס הנתונים (SQL)
יש לפתוח את ה-SQL Server, להריץ את סקריפט ה-SQL ליצירת בסיס הנתונים והטבלאות (`EventStore`).
לאחר מכן, יש לעדכן את מחרוזת החיבור (**Connection String**) בקובץ `appsettings.json` תחת פרויקט `WebApp`.

### 2. הרצת ה-Backend (C#)
1. פתחי את ה-Solution ב-Visual Studio.
2. הגדירי את פרויקט **`WebApp`** כ-Startup Project (קליק ימני על WebApp -> Set as Startup Project).
3. לחצי על כפתור ה-**Play** או לחצי על `F5`.
4. השרת יעלה ויקשיב בכתובת המאובטחת (למשל: `https://localhost:7003`).

### 3. הרצת ה-Frontend (Angular)
1. פתחי את הטרמינל בתיקיית ה-Angular (`angularProject`).
